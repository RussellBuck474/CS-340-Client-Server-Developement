# CS-340-Client-Server-Developement
This is for Client server development class for SNHU,  
1. How do you write programs that are maintainable, readable, and adaptable?
Writing maintainable and adaptable programs starts with organizing code into clear, reusable components. In Project One, the CRUD Python module was designed as a standalone class that handled all database interactions. This separation made the code easier to read and prevented duplication across the dashboard in Project Two. The advantage of this approach is that when the database logic needs to change, it only has to be updated in one place. The same CRUD module could be reused in future dashboards, APIs, or automation scripts because it abstracts the database operations behind clean, predictable methods.

2. How do you approach a problem as a computer scientist?
I approach problems by breaking them down into smaller, logical steps and identifying the requirements before writing any code. For the Grazioso Salvare project, I analyzed the database structure, the dashboard requirements, and the user interactions needed to filter and display data. Compared to earlier courses, this project required more planning because it involved connecting multiple systems: MongoDB, Python, and Dash. In the future, I would continue using this structured approach—starting with understanding the client’s needs, designing the data model, and then building the interface around it. This ensures the final product is both functional and aligned with the client’s goals.

3. What do computer scientists do, and why does it matter?
Computer scientists solve problems by designing systems that make information easier to access, understand, and use. In a project like this, the dashboard helps a company such as Grazioso Salvare quickly identify dogs suitable for different types of rescue work. Instead of manually searching through thousands of records, the dashboard automates the process and presents results instantly. This improves efficiency, reduces errors, and supports better decision‑making. Work like this matters because it allows organizations to focus more on their mission—in this case, saving lives—while technology handles the data processing behind the scenes.****
